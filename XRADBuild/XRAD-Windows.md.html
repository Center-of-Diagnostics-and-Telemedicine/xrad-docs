Создать переменную среды можно с командной строки: ~~~~ none setx XRADThirdParty "Q:\3rd_party" ~~~~ # Сборка XRAD # 1. Создайте папку для исходных текстов библиотеки XRAD. Имя может быть любое. В качестве примера будем использовать имя `Q:\XRAD`. Создайте переменную среды `XRADRoot`, указывающую на папку `Q:\XRAD`. Создать переменную среды можно с командной строки: ~~~~ none setx XRADRoot "Q:\XRAD" ~~~~ 1. Клонируйте в `Q:\XRAD` репозиторий XRAD. 1. Переключитесь на вершину ветки `develop`. 1. Создайте папку для исходных текстов проекта XRADLibrariesBuild. Имя может быть любое. В качестве примера будем использовать имя `Q:\XRADPrograms\XRADLibrariesBuild`. 1. Клонируйте в `Q:\XRADPrograms\XRADLibrariesBuild` репозиторий XRADLibrariesBuild. 1. Переключитесь на вершину ветки `develop`. 1. Если это не было сделано ранее, создайте папку для установки общих библиотек XRAD. Имя может быть любое. В качестве примера будем использовать имя `Q:\XRADInstall`. Создайте переменную среды `XRADInstall`, указывающую на папку `Q:\XRADInstall`. Создать переменную среды можно с командной строки: ~~~~ none setx XRADInstall "Q:\XRADInstall" ~~~~ 1. Откройте solution для сборки проекта: - XRADLibrariesBuild.sln для MSVC 2015 - XRADLibrariesBuild-2019.sln для MSVC 2019 1. Соберите solution в нужных конфигурациях. # Сборка приложений # TODO Для сборки в MSVC 2019 используются проекты и solution с суффиксом `-2019`. # Устройство системы сборки приложений с библиотекой XRAD # TODO К приложениям подключаются следующие библиотеки: - Консольне приложения: XRADConsoleUI - Приложения GUI: XRADGUI - Модули dll: XRADSystem - Приложения, модули, работающие с DICOM, дополнительно подключают библиотеки: - без GUI: XRADDicom - с GUI: XRADDicomGUI Подключение библиотек осуществляется подключением property sheet _после_ `$(UserRootDir)\Microsoft.Cpp.$(Platform).user.props`. В файле проекта соответствующие теги имеют вид: - MSVC 2015 ~~~~ xml <Import Project="$(XRADRoot)\XRADConsoleUI\MSVC\XRADConsoleUI.props" /> <Import Project="$(XRADRoot)\XRADGUI\MSVC\XRADGUI.props" /> <Import Project="$(XRADRoot)\XRADSystem\MSVC\XRADSystem.props" /> <Import Project="$(XRADRoot)\XRADDicom\MSVC\XRADDicom.props" /> <Import Project="$(XRADRoot)\XRADDicom\MSVC\XRADDicomGUI.props" /> ~~~~ - MSVC 2019 ~~~~ xml <Import Project="$(XRADRoot)\XRADConsoleUI\MSVC\XRADConsoleUI-2019.props" /> <Import Project="$(XRADRoot)\XRADGUI\MSVC\XRADGUI-2019.props" /> <Import Project="$(XRADRoot)\XRADSystem\MSVC\XRADSystem-2019.props" /> <Import Project="$(XRADRoot)\XRADDicom\MSVC\XRADDicom-2019.props" /> <Import Project="$(XRADRoot)\XRADDicom\MSVC\XRADDicomGUI-2019.props" /> ~~~~ Property sheet библиотек DICOM должны включаться после property sheet основных библиотек XRAD. Порядок включение важен. В среде MSVC эти property sheet подключаются в каждой конфигурации и платформе в окне Property Manager. В списке в окне они отображаются в порядке, обратном порядку включения. Они должны быть выше `Microsoft.Cpp.$(Platform).user.props`. Чтобы подключить проекты сборки библиотек XRAD в solution проекта приложения (это делать необязательно), нужно использовать в пути переменную среды `XRADRoot`. Сделать это через GUI, вероятно, невозможно. После подключения проектов библиотек нужно отредактировать файл sln в текстовом редакторе. В файле sln подключение выглядит следующим образом: ~~~~ none Project("{8BC9CEB8-8B4A-11D0-8D11-00A0C91BC942}") = "XRADDicomGUI", "%XRADRoot%\XRADDicomGUI\MSVC\XRADDicomGUI.vcxproj", "{6A6EEC8F-C1AB-445E-A73B-5B4368599D9B}" ... ~~~~ В sln библиотеки следует подключать с зависимостями: - XRADGUI: XRADSystem, XRADBasic - XRADConsoleUI: XRADSystem, XRADBasic - XRADDicom: XRADSystem, XRADBasic - XRADDicomGUI: XRADDicom, XRADGUI, XRADSystem, XRADBasic
